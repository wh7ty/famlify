# Famlify Project Instructions

## Project Memory

Always use `mind.md` as the project diary for this repository.

Before meaningful work:

- Read `mind.md` first.
- Use it to recover project context, decisions, open questions, and next steps.

After meaningful work:

- Update `mind.md` when a decision, implementation step, design direction, blocker, or next step changes.
- Keep entries short, practical, and current.

This rule is mandatory for work in this project. Treat `mind.md` as the local memory skill for Famlify.

## Design System

Always use `docs/design-system.md` as the local design-system guide for Famlify UI work.

Before UI work:

- Read `mind.md`.
- Read `docs/design-system.md`.
- Apply the design system consistently unless the user explicitly changes direction.

After meaningful UI decisions:

- Update `docs/design-system.md` when tokens, component patterns, responsive rules, or visual direction change.
- Update `mind.md` with the decision and next step.

## Current Product Direction

- Private web app for the user and his girlfriend only.
- First build a responsive prototype.
- Mobile and tablet matter most; desktop should remain clean and usable.
- Start with a "Heute" dashboard for shared daily life.
